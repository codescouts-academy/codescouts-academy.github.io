Mob programming

What is it?

As mobprgramming.org states:

‚ÄúAll the brilliant people working on the same thing, at the same time, in the same place and at the same computer‚Äù

And as the name suggests, Mob Programming is an approach in software development where the whole team works on the same thing. This is similar to pair-programming where two people sit at the same computer and work on a task or a problem at the same time. One usually does the coding and the other ‚Äònavigates‚Äô.

Mob programming is not restricted to any kind of work, it can be used for:

  • User Stories
  • Requirements
  • Testing
  • Designing
  • Bug Fixing
  • Trouble Shooting
  • Testing

This can mean that a typical mob will design, develop, test and deploy all together.

Benefits of mob programming

  • Increased learning and communication
  • Increased capacity to create
  • Increased quality of code
  • Happier and more cohesive team

Roles in mob programming

  • The driver
  • The navigator
  • The mob
  • The facilitator
